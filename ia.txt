
PROJETO: Sistema de Gestão - Dubom Refrigeração
DATA DA ÚLTIMA ATUALIZAÇÃO: Junho 2026

--- TECNOLOGIAS ---
- Backend: PHP Puro (Arquitetura Híbrida: MVC Básico + Arquivos Legados em migração).
- Database: SQLite (Consolidado).
- Frontend: HTML5, Bootstrap 5, JavaScript (Vanilla).
- Autenticação: Sessão PHP com controle de nível de acesso (Admin/Funcionário).
- Versionamento & Deploy: Git, GitHub e Hostinger (Deploy via Git).

--- ARQUIVOS PRINCIPAIS (MVC) ---
1. src/Controllers/HomeController.php: Dashboard, Kanban de OS e Criação de Serviços.
2. src/Controllers/ClienteController.php: Gestão de Clientes (CRUD).
3. src/Controllers/ProdutoController.php: Gestão de Estoque e Peças.
4. src/Controllers/FornecedorController.php: Gestão de Fornecedores.
5. src/Controllers/UsuarioController.php: Gestão de Acesso e Equipe.
6. src/Controllers/CatalogoController.php: Gestão de Serviços e Preços Padrão.
7. src/Views/*.view.php: Views correspondentes aos controladores acima.

--- ARQUIVOS DE SUPORTE ---
1. public/index.php: Front Controller.
2. src/Router.php: Roteador simples (Regex).
3. conexao.php: Configuração do Banco de Dados e Migrations Automáticas.

--- REGRAS DE NEGÓCIO IMPLEMENTADAS ---

1. NÍVEIS DE ACESSO (RBAC):
   - Admin: Acesso total (Financeiro, Usuários, Fornecedores, Exclusão).
   - Funcionário: Acesso operacional (Visualizar/Criar OS, Clientes).

2. FLUXO DE CAIXA E ESTOQUE:
   - Produtos: Controle de quantidade e preço de custo/venda.
   - Serviços: Composição de valor (Mão de Obra + Peças).
   - Financeiro: Status de pagamento (Pendente/Pago) e cálculo de lucro estimado.

3. UX/UI (AGILIDADE):
   - Autocomplete Inteligente: Sugere clientes e serviços ao digitar.
   - Cadastro Rápido Inteligente: Se o cliente não existe na criação da OS, o sistema avisa e exibe campos opcionais (telefone, endereço) para um cadastro mais completo na mesma tela.
   - Kanban: Visualização de status por colunas (Agendado -> Em Andamento -> Concluído -> Pago).
   - Visualização Moderna de OS: Nova tela de visualização/impressão com link direto para WhatsApp e PDF.

--- CONTEXTO DA EMPRESA (PERSONA) ---
- Equipe: Cresceu para 4 pessoas. O dono (Pai) não trabalha mais sozinho.
- Operação: O dono sabe endereços de cabeça, mas a equipe precisa dessa informação no sistema.
- Comunicação Atual: O dono envia áudios no WhatsApp explicando o serviço e valor. Funciona pela confiança, mas o objetivo é profissionalizar com Recibos/OS formais.
- Futuro: Venda de peças e controle de estoque/despesas.

--- ESTRUTURA DE DADOS (SQLite) ---
Principais Tabelas:
- servicos (OS, Status, Valores)
- servicos_itens (Detalhes da OS)
- clientes (Dados PF/PJ)
- produtos (Estoque)
- fornecedores (Parceiros)
- usuarios (Login)
- catalogo (Preços padrão de serviços)

--- COMO ENVIAR ATUALIZAÇÕES PARA O GITHUB ---
1. Abra o terminal na pasta do projeto.
2. Inicialize o Git:
   git init
3. Adicione os arquivos:
2. Verifique o status (opcional):
   git status
3. Adicione todas as mudanças:
   git add .
4. Faça o primeiro commit:
   git commit -m "Primeiro commit: Sistema Dubom"
5. Crie o repositório no site do GitHub (sem README/gitignore).
6. Conecte o repositório remoto (substitua a URL):
   git remote add origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git
7. Envie os arquivos:
   git push -u origin main
